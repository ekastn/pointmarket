Tentang NLP API Tester

NLP API Tester adalah alat diagnostik yang memungkinkan Anda menguji berbagai API NLP pada sistem POINTMARKET tanpa perlu berinteraksi dengan halaman PHP. Alat ini membantu mengidentifikasi apakah masalah terletak pada frontend, backend, atau keduanya.

Cara Mengakses NLP API Tester

Untuk mengakses NLP API Tester, ikuti langkah-langkah sederhana berikut ini:

Langkah-langkah Akses
  1. Buka browser web Anda (Chrome, Firefox, Edge, atau browser lainnya)
  2. Ketikkan URL berikut di address bar:
    http://localhost/pointmarket/nlp-api-tester.html
  3. Tekan Enter untuk membuka halaman tester
Kelebihan Versi Ini

Tool ini tidak memerlukan login atau setup khusus. Langsung bisa digunakan!

Struktur Halaman Tester

Halaman tester terdiri dari tiga bagian utama yang mudah dipahami:

Test API Endpoints

Untuk pengujian cepat API yang tersedia

Send Custom Request

Untuk mengirim request khusus dengan parameter yang dikustomisasi

API Information

Informasi tentang endpoint yang tersedia dan tips pengujian

Pengujian API Endpoints

Bagian ini memungkinkan Anda untuk melakukan pengujian cepat terhadap tiga API yang tersedia. Setiap API memiliki fungsi berbeda untuk membantu Anda mendiagnosis masalah:

1. Test Main API

Fungsi: Menguji API utama (nlp-analysis.php) untuk memverifikasi fungsionalitasnya.

Langkah-langkah:
  1. Klik tombol "Test Main API"
  2. Sistem akan mengirim request GET dengan parameter test=1
  3. Hasil akan ditampilkan di bagian "Results" di bawah tombol
Hasil yang diharapkan: Status 200 dengan JSON valid yang berisi informasi tentang status API.
2. Test Backup API

Fungsi: Menguji API backup (nlp-backup-api.php) yang menggunakan model NLP dasar.

Langkah-langkah:
  1. Klik tombol "Test Backup API"
  2. Sistem akan mengirim request GET dengan parameter test=1
  3. Hasil akan ditampilkan di bagian "Results" di bawah tombol
Hasil yang diharapkan: Status 200 dengan JSON valid yang berisi informasi tentang status API backup.
3. Test Mini API

Fungsi: Menguji API mini (nlp-mini-api.php) yang selalu mengembalikan data dummy valid.

Langkah-langkah:
  1. Klik tombol "Test Mini API"
  2. Sistem akan mengirim request GET dengan parameter test=1
  3. Hasil akan ditampilkan di bagian "Results" di bawah tombol
Hasil yang diharapkan: Status 200 dengan JSON valid yang berisi data dummy. API ini selalu seharusnya berfungsi karena tidak memiliki dependensi eksternal.
Tips Untuk Pemula:

Mulailah dengan menguji Mini API. Jika ini berhasil, Anda tahu bahwa konektivitas dasar berfungsi, dan masalah mungkin terletak pada API utama atau backup.

Urutan yang disarankan: Mini API → Backup API → Main API

Mengirim Request Kustom

Bagian "Send Custom Request" memungkinkan Anda untuk membuat request yang disesuaikan dengan kebutuhan spesifik Anda. Fitur ini cocok untuk pengguna yang ingin menguji skenario khusus:

Langkah 1: Pilih API dan Method
Pilih API Endpoint dari dropdown:
  • Main API (nlp-analysis.php)
  • Backup API (nlp-backup-api.php)
  • Mini API (nlp-mini-api.php)
  • Debug API (nlp-api-debug.php)
Pilih Request Method:
GET
Statistik, tes, parameter
POST
Kirim teks untuk analisis
Langkah 2: Isi Parameter

Tergantung pada method yang dipilih, Anda perlu mengisi:

Untuk Request GET:
action=statistics&v=12345
Parameter umum: action, test, v (cache busting)
Untuk Request POST:
{"text": "Ini adalah contoh teks untuk dianalisis."}
Selalu gunakan format JSON yang valid

Klik untuk mengirim request
Penting untuk Diingat!

Pastikan format JSON valid saat menggunakan method POST. JSON yang tidak valid akan menyebabkan error pada API. Gunakan tools online JSON validator jika diperlukan.

Troubleshooting Masalah Umum

1. API Mengembalikan Error 401 Unauthorized

Kemungkinan Penyebab:

  • Session login tidak ada atau tidak valid
  • API memeriksa status login sebelum memproses request

Solusi:

  1. Gunakan file get-session-info.php untuk memeriksa status session
  2. Gunakan file create-test-session.php untuk membuat session pengujian
  3. Edit file API untuk menonaktifkan sementara pengecekan login untuk debugging
2. API Mengembalikan HTML Alih-alih JSON

Kemungkinan Penyebab:

  • Error PHP yang muncul sebelum header JSON ditetapkan
  • Include file yang gagal atau error syntax

Solusi:

  1. Periksa response HTML untuk mengidentifikasi error PHP
  2. Tambahkan error handling di awal file API
  3. Gunakan Mini API sebagai alternatif sementara

Contoh error yang sering muncul:

<b>Warning</b>: require_once(includes/nlp-model.php): Failed to open stream: No such file or directory in <b>/path/to/api/nlp-analysis.php</b> on line <b>10</b>
3. Network Error atau CORS Issue

Kemungkinan Penyebab:

  • Konfigurasi server yang tidak benar
  • Headers CORS tidak ditetapkan dengan benar

Solusi:

  1. Periksa console browser (F12) untuk detail error
  2. Pastikan headers CORS ditetapkan dengan benar di file API
  3. Pastikan server web (Apache/XAMPP) berjalan dengan benar

Headers CORS yang diperlukan:

header('Access-Control-Allow-Origin: *'); header('Access-Control-Allow-Methods: POST, GET, OPTIONS'); header('Access-Control-Allow-Headers: Content-Type');

Strategi Debugging yang Efektif

Langkah-langkah Sistematis
  1. Verifikasi Konektivitas Dasar
    • Klik "Test Mini API"
    • Jika berhasil, konektivitas dasar berfungsi
    • Jika gagal, kemungkinan ada masalah dengan server atau konfigurasi web
  2. Periksa API Utama
    • Klik "Test Main API"
    • Bandingkan dengan hasil Mini API
    • Jika Main API gagal tapi Mini API berhasil, masalahnya ada pada API utama
  3. Coba Request POST Sederhana
    • Gunakan bagian "Send Custom Request"
    • Pilih Mini API dengan method POST
    • Kirim teks sederhana untuk dianalisis
    • Verifikasi bahwa response berformat JSON yang valid
  4. Debug API Utama dengan Detail
    • Ulangi langkah 3 tapi gunakan Main API
    • Periksa error yang muncul
    • Gunakan informasi error untuk mengidentifikasi masalah spesifik
